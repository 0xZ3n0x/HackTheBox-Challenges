# Hamming(7,4) Statistical Signal Decoder

**Challenge:** HM74

**Category:** Hardware

**Platform:** [HackTheBox](https://www.hackthebox.com/)

This repository contains a C++ solution for the **HM74** challenge. The objective is to recover a hidden flag from a noisy bitstream originally generated by a SystemVerilog-based transmitter.

---

## 1. Challenge Analysis

### Signal Encoding Logic

The challenge is based on a **Hamming(7,4)** error-correction scheme. By analyzing the logic of the hardware encoder, the parity mapping and bit order were identified as:

* **Data Bits ($d$):** A 4-bit input nibble ($d_3, d_2, d_1, d_0$).
* **Parity Bits ($p$):** * $p_0 = d_3 \oplus d_2 \oplus d_0$
* $p_1 = d_3 \oplus d_1 \oplus d_0$
* $p_2 = d_2 \oplus d_1 \oplus d_0$


* **Serialized Output:** The 7-bit transmission block follows the format: `{p0, p1, d3, p2, d2, d1, d0}`.

### Data Capture Characteristics

The input data consists of a large set of "Captured" bitstreams. Due to simulated environmental noise, many packets contain bit flips. While standard Hamming correction handles **1-bit errors** per 7-bit block, this solution employs statistical analysis to ensure 100% accuracy even when multi-bit errors occur in individual packets.

---

## 2. Solution Implementation

The solution implements a **Frequency Analysis** (statistical majority vote) approach to reconstruct the most probable flag.

### Hamming Decoding

The `HammingDecoder` processes each 7-bit block by:

1. Calculating the **Syndrome** ($s_0, s_1, s_2$) based on the XOR logic identified in the encoder.
2. Identifying the error bit position (if any) and flipping it back to its original state.
3. Extracting the 4-bit data nibbles ($d_3, d_2, d_1, d_0$).

### Denoising via Frequency Queue

Because the noise level can exceed the correction capability of a single Hamming block, we utilize a **Max-Heap** based `FrequencyQueue`:

* **Tracking:** Every nibble position in the flag is tracked across the entire set of captured packets.
* **Statistical Selection:** The most frequent nibble at each specific index is selected as the ground truth.
* **Reconstruction:** Pairs of 4-bit nibbles are combined to form 8-bit ASCII characters.

## 5. Summary of Logic

The flag is reconstructed by joining the two most-frequent 4-bit nibbles at each position:

$$char = (HighNibble \ll 4) \lor LowNibble$$

This robust approach allows for successful flag extraction even in highly unstable transmission environments where traditional single-packet decoding would fail.
